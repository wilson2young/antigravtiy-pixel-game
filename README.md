# PIXEL QUEST - 像素风闯关问答游戏

## 专案简介
本专案是一个基于 React + Vite 开发的怀旧 2000 年代街机像素风问答游戏。前端透过环境变数设定题目与通关门槛，游戏内预设使用 DiceBear API 作为关卡“关主”的像素头像。
后端伺服器完全采用免费轻量的 **Google Sheets + Google Apps Script (GAS)** ，达成题库随机抽取、成绩即时计算与远端资料库留存的功能。

## 后端资料库设定教学 (Google Sheets & Apps Script)

### 步骤一：建立 Google Sheets 资料库
1. 登入 Google 帐号，建立一个全新的空白 **Google 试算表 (Google Sheets)**。
2. 将底部的第一个工作表改名为 **题目**。
3. 在第一列（A1 到 G1）分别填入以下确切的栏位名称（表头）：
   `题号` | `题目` | `A` | `B` | `C` | `D` | `解答`
4. 添加一个新的工作表，命名为 **回答**。
5. 在**回答**的第一列（A1 到 G1）分别填入以下表头栏位名称：
   `ID` | `闯关次数` | `总分` | `最高分` | `第一次通关分数` | `花了几次通关` | `最近游玩时间`

### 步骤二：挂载 Google Apps Script (GAS)
1. 在刚刚建好的试算表顶端选单中，点选 **扩充功能 (Extensions)** -> **Apps Script**。
2. 这会开启一个新的专属代码编辑专案。清除编辑器内预设的 `myFunction`。
3. 将我们专案提供的 `Code.gs` 后端程式码，**完整复制并贴上**到编辑器中。
4. 按下右上角的“储存档案”图示（或直接使用快捷键 `Ctrl + S` / `Cmd + S`）。

### 步骤三：发布为网页应用程式 (Web App)
这一步最重要，只有部署并获得 URL 后，前端才能呼叫 API：
1. 在 Apps Script 编辑器的右上角，点击蓝色的 **部署 (Deploy)** 按钮，选择 **新增部署作业 (New deployment)**。
2. 左上角点击“小齿轮”图示，选择类型为 **网页应用程式 (Web app)**。
3. 设定以下权限：
   - **执行身份**：设定为您自己（`我 (你的 Email)`）。
   - **谁可以存取**：设定为 **所有人 (Anyone)**。
4. 点击蓝色的 **部署** 按钮。
5. **授权步骤**（首次部署必经流程）：
   - 画面会跳出“需要授权”的视窗，请点选「审查权限」。
   - 选择你的 Google 帐号登入。
   - 没过安全验证此时会看见 Google 红色警告“Google 尚未验证这个应用程式”。
   - 点击左下角的「**进阶 (Advanced)**」，接着点选「前往该专案 (不安全)」。
   - 点击最下方的「允许」。
6. 完成部署后，拷贝画面上以 `https://script.google.com/macros/s/.../exec` 结尾的 **网页应用程式网址 (URL)**，请妥善保存。

## 本地端环境变数设定 (Vite)
前端部分需要将刚刚的网址挂载到环境变数中，才能将通讯接上。

1. 在专案根目录（与 `package.json` 同级）下建立 `.env` 档案。
2. 将以下变数贴入，并把网址替换为你刚才复制的 GAS URL：
```env
VITE_GOOGLE_APP_SCRIPT_URL=https://script.google.com/macros/s/你的_ID_贴在这里/exec
VITE_PASS_THRESHOLD=3
VITE_QUESTION_COUNT=5
```
3. 开启终端机，输入 `npm install` 确保包体更新。
4. 输入 `npm run dev` 启动专案，你的浏览器会在 `http://localhost:5173` 运行游戏！

## GitHub Pages 自动部署教学 (GitHub Actions)

本专案已经内建了自动部署至 GitHub Pages 的 CI/CD 流程 (`.github/workflows/deploy.yml`)。只需将这包程式码推送到你的 GitHub 储存库，并设定好环境变数 Secrets，每次推送至 `main` 或 `master` 分支时便会自动发布最新版本的游戏页面。

### 部署步骤

1. **建立 GitHub 储存库 (Repository)**：在 GitHub 建立一个新储存库，并将本专案代码推送到该储存库的主分支。
2. **设定 Secrets 环境变数**：
   - 进入储存库页面的 **Settings (设定)**。
   - 在左侧选单点开 **Secrets and variables** -> 点击 **Actions**。
   - 点击绿色按钮 **New repository secret**，依序新增以下三个密钥（值请填入你准备好的参数以对应 `.env.example`）：
     - `VITE_GOOGLE_APP_SCRIPT_URL`：填上你的 GAS 网页应用程式部署 URL。
     - `VITE_PASS_THRESHOLD`：填入过关门槛（例如 `3`）。
     - `VITE_QUESTION_COUNT`：填入抽卡题数（例如 `5`）。
3. **启用 GitHub Pages 工作流**：
   - 前往储存库页面的 **Settings**。
   - 左侧选单点击 **Pages**。
   - 在 **Build and deployment** 区块内的 **Source** 选项，下拉选择 **GitHub Actions**。

设定完毕后，你可以到上方的 **Actions** 页签查看部署进度。日后每次你推送代码更新时，系统皆会自动透过 Actions 帮你完成构建并部署到属于你的 GitHub Pages 网址！
